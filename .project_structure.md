## ğŸš€ Common Development Workflows

### Adding a New Feature

1. **Identify component**: Use lookup table above
2. **Check existing code**: Look for similar implementations
3. **Add functionality**: Modify appropriate files
4. **Update interfaces**: Add ROS2 messages if needed
5. **Write tests**: Add unit/integration tests
6. **Update docs**: Modify READMEs and docs

### Debugging Issues

1. **Check logs**: `ros2 topic echo /rosout`
2. **Verify topics**: `ros2 topic list | grep <component>`
3. **Test nodes**: `ros2 node list | grep <component>`
4. **Check config**: `ros2 param list /<node_name>`
5. **Run tests**: `python -m pytest tests/unit/test_<component>.py`

### Deployment Process

1. **Build packages**: `colcon build --symlink-install`
2. **Source environment**: `source install/setup.bash`
3. **Launch system**: `./start.py prod autonomy`
4. **Monitor health**: Check dashboard and logs
5. **Validate**: Run integration tests
