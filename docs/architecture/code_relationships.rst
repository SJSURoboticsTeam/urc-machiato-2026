Understanding Code Relationships
=================================

Import Dependencies
-------------------

Import dependency graphs show:

- **Direct dependencies**: Modules that are directly imported
- **Transitive dependencies**: Indirect relationships through other modules
- **Circular dependencies**: Potential design issues that should be avoided
- **Layer violations**: When high-level modules import low-level ones inappropriately

Class Hierarchies
-----------------

Class hierarchy diagrams reveal:

- **Inheritance patterns**: How classes extend and specialize behavior
- **Interface implementations**: Which classes implement specific interfaces
- **Polymorphism opportunities**: Where different implementations can be swapped
- **Design patterns**: Recognition of common patterns like Strategy, Factory, etc.

Automated Analysis Benefits
===========================

These diagrams are automatically generated by analyzing the actual codebase:

1. **Always Current**: Diagrams reflect the current state of the code
2. **Version Controlled**: Changes in code structure are automatically reflected in docs
3. **Refactoring Guide**: Helps identify areas that need attention during refactoring
4. **Documentation Maintenance**: Reduces manual effort to keep architecture docs current

Analysis Scope
==============

The analysis includes:

- **Python modules**: All .py files in the specified directories
- **Import statements**: Both absolute and relative imports
- **Class definitions**: Inheritance relationships and base classes
- **Package structure**: How modules are organized into packages

Excluded from analysis:

- **Test files**: test_*.py and files in test directories
- **Cache files**: __pycache__ directories and .pyc files
- **External dependencies**: Standard library and third-party packages
- **Generated files**: Auto-generated or compiled files
